Updated: 2026-05-09

## Summary

This archive records the investigation of the OBB real-time profile that opened but processed 0 frames, the board-side binary rebuild/deployment, and the remaining blocker. ## PC Runtime Preflight Change

Touched file:

```text
tools/adb_imx415_rknn_live_view.py
```

Added board runtime file preflight before starting the remote process. The script now checks that these required files exist:

- remote binary;
- main model;
- two-stage defect model.

When a required file is missing, the script prints:

```text
Missing board runtime file(s):
```

and exits cleanly, avoiding the previous failure mode where the stale board process could hit `fopen ... fail`, crash, and only show `Processed frames: 0`.
