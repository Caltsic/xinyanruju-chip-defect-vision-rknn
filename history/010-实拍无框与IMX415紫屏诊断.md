# 实拍无框与 IMX415 紫屏诊断

更新时间：2026-05-03

## 现象

- `captures/practice` 中的 6 张图片是 OpenCV 实时窗口截图，不是纯相机原始帧。
- 截图状态栏均显示 `det 0`。
- 实时窗口偶尔出现局部紫屏。
- 后续短帧诊断时，板端实时程序出现 `camera dequeue timeout`，无法从 `/dev/video42` 取得帧。

## 模型侧结论

使用 `chipcheck_yolov8_detect.onnx` 在 6 张实拍截图上做离线推理：

- `conf=0.25`：6 张均为 0 框。
- `conf=0.05`：只有少量低置信度大框，最高约 `0.19`，不具备可靠检测意义。
- 同一 ONNX 在训练集测试图抽样上能正常输出高置信度框，说明模型文件和基本输出格式正常。

判断：当前无框不是 PC 端绘制或 RKNN 协议解析单独导致的。主要原因是实拍画面与训练集分布差异大：训练集偏近距离灰度缺陷图，实拍是 960x540 彩色整芯片/元件视角，且不一定存在训练类别中的缺陷。

## 实时解析侧结论

当前板端 RKNN INT8 模型信息正常：

```text
input:  images, dims=[1, 640, 640, 3], type=INT8
output: output0, dims=[1, 8, 8400], type=INT8
```

PC 端 `tools/adb_imx415_rknn_live_view.py` 已增强诊断显示：

- 状态栏从 `det N` 改为 `det raw/drawn`。
- `raw` 是板端 RYL1 协议返回的原始框数。
- `drawn` 是 PC 端归一化/裁剪后实际绘制的框数。
- 新增 `--save-clean-snapshot`，可保存不带状态栏和框的干净解码帧。

若后续窗口显示 `det 0/0`，表示板端后处理本身没有返回框；若显示 `det 3/0`，才说明 PC 端绘制/坐标过滤有问题。

## 相机与紫屏结论

截图本身未发现典型 NV12/NV21 错解或 B/R 通道互换。6 张截图的 B/G/R 均值接近，紫色像素占比很低。

但板端后续短帧测试出现：

```text
camera dequeue timeout
MIPI_CSI2 ERR2:0x10000000
```

独立 `v4l2-ctl` 对 `/dev/video42` 做 stream 测试也无法得到帧，输出文件 0 字节。`/dev/video42` 格式为 `960x540 NV12`，`bytesperline=960`，stride 与当前 C++ 拷贝逻辑匹配。

判断：紫屏更像是 IMX415 到 RK3576 的 MIPI/CSI 接收链路偶发错误或传感器流不稳定，不像当前 PC 端 NV12 转 BGR 代码导致。若紫屏继续出现，应优先检查排线、接口压接、供电、摄像头模组固定和 ISP/3A 服务状态。

## 新增工具

离线 ONNX 诊断：

```powershell
F:\anaconda\python.exe .\tools\diagnose_chip_defect_onnx.py
```

只打印不保存：

```powershell
F:\anaconda\python.exe .\tools\diagnose_chip_defect_onnx.py --no-save
```

保存干净相机帧与带框帧：

```powershell
F:\anaconda\python.exe .\tools\adb_imx415_rknn_live_view.py --frames 20 --headless `
  --save-snapshot .\captures\practice\diag_annotated.jpg `
  --save-clean-snapshot .\captures\practice\diag_clean.jpg
```

低阈值观察弱候选：

```powershell
F:\anaconda\python.exe .\tools\adb_imx415_rknn_live_view.py --conf 0.05
```

FP 基线对比：

```powershell
F:\anaconda\python.exe .\tools\adb_imx415_rknn_live_view.py --remote-model model/chipcheck_yolov8_detect_fp.rknn
```

## 下一步

1. 先恢复相机出帧：检查 IMX415 排线/接口，必要时重启板子；恢复后用 `--save-clean-snapshot` 抓纯净帧。
2. 用真实缺陷、近距离、满画幅的芯片表面/引脚场景复测；不要用普通完好芯片本体视角判断缺陷模型。
3. 如果真实缺陷仍无框，采集 100-300 张当前镜头、当前光照、当前距离下的样本补充标注，做一次小规模增量训练或重新量化校准。
4. 若只是想先检测“芯片本体位置”，需要另训一个芯片/元件目标检测模型；当前模型只训练了缺陷类别。
