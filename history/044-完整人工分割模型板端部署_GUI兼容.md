## Scope

本次任务是把完整人工标定后训练并转换得到的 YOLOv8s-Seg INT8 RKNN 模型部署到泰山派 RK3576 板端，并保持现有 GUI / 实时脚本兼容。

关键决策：
- 不修改本地 GUI 代码。
- 不修改实时脚本代码。
- 通过覆盖板端默认分割缺陷模型路径完成兼容：

```text
/userdata/rknn_yolo11_demo/model/chipcheck_yolov8_seg_split_int8.rknn
```

这意味着 GUI 只要继续勾选 `Use segmentation defect model`，就会沿用现有 `chip-two-stage-seg-imx678` 运行链路加载新模型。

## GUI And Runtime Defaults

`chip-two-stage-seg-imx678` 默认使用：

```text
remote binary: ./rknn_chip_two_stage_maixcam_stream
chip ROI model: model/chip_roi_yolov8_detect_split_int8.rknn
defect seg model: model/chipcheck_yolov8_seg_split_int8.rknn
```

类别标签注意事项：

```python
CHIP_TWO_STAGE_CLASSES = ["chip", "ZF-scratch", "scratch", "broken", "pinbreak"]
```

当前 GUI / 实时脚本默认类别主要由上述内置 `CHIP_TWO_STAGE_CLASSES` 驱动。板端也推送了 label 文件用于留档和后续显式传参场景，但默认 GUI 路径不依赖该文件。
