# GUI缺陷模型显式切换最新版并新增原图显示

Updated: 2026-05-11

## Task

Update GUI defect recognition to explicitly use the latest segmentation defect model, and add an original-frame display mode.

This applies to:

- local Qt GUI
- local OpenCV GUI
- board-side GUI files under `/userdata/chipcheck_vision/tools/chip_capture_gui`

## Problem

Before this update, GUI segmentation mode functionally used the latest board model only because the generic board filename had already been replaced.

That was ambiguous:

```text
model/chipcheck_yolov8_seg_split_int8.rknn
```

The GUI code was still pointing to a generic filename rather than explicitly binding to the latest deployed segmentation artifact.

The user also requested a clear original-image display mode in GUI.

## Explicit Latest Defect Model

The GUI now explicitly uses:

```text
model/chipcheck_yolov8s_seg_full_manual_plus_project4_20260510_ft_split_int8.rknn
```

This was added as:

```text
LATEST_DEFECT_SEG_REMOTE_MODEL
```

in:

```text
tools/chip_capture_gui/settings.py
```

Helper added:

```text
default_defect_model_for_profile(profile, defect_model_kind)
```

This ensures segmentation GUI paths no longer rely on the generic default filename to infer “latest”.
